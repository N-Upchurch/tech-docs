What is a CSS Variable

  • CSS variables allow you to reuse values throughout a document.

Variables vs. Custom Properties

Pedantic fuddy-duddies will tell you that they're not variables; they're CsS CuStOm PrOpErTiEs. The fact is, they work in a slightly different manner, but for all intents and purposes, they're variables.

Why use the bastards?

Pfft. Why not? Change multiple elements by editing one line, avoid errors, cut down on code repetition. It's 2019, and we're living the dream.

Basic Usage

  • Define variables in :root for good times.

Setup

Slap your variables in the :root element, like this:

    :root {

        --my-variable: #ffffff;

    }

Usage

All you need to do is throw your variable in the ring as a value like this:

    body {

        color: var(--my-variable);

    }

These bad boys can only be used for property values. Don't try to store entire CSS rules in them; it'll not work.

    :root {

        /* This works */
        --color: #ffffff;

        /* This doesn't */
        --color: color: #ffffff;

    }

Syntax

Adding a fallback is as easy as:

    body {
    
        color: var(--my-color, white);
    
    }

Fallback Nesting Syntax

Wanna fall back to another variable, and then an alternative, for some reason? Do it like this:

    body {
    
        color: var(--color-1, var(--color-2, white))
    
    }

Javascript

  • Access CSS variables with JS

Get and set CSS variables like so:

    // get variable from inline style
    element.style.getPropertyValue("--my-var");

    // get variable from wherever
    getComputedStyle(element).getPropertyValue("--my-var");
